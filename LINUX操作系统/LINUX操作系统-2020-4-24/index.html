LINUX操作系统--第九节(编辑器 vim & 用户管理)

发表于 2020-04-24

题目

  1. 考虑下述文本，我们希望能够在第一列的数字后面增加一个.，假设已经高亮选择了相应的行，请使用normal命令实现
    1 tony 100
    2 tom 90
    3 alice 99
    4 bob 80

  2. 假设你正在编辑一个python源程序，光标正在一个函数的头部所在的行，你发现在这个函数里面某个变量的名字写错了，变量名varr要改成var,请将该函数中的所有varr改成var。如何确定哪些行属于这个函数呢？假设该函数后面还有一个函数，该函数的行首以def后跟一个空格开始，这样从当前行开始，到下一个def语句所在行的前一行都认为属于当前的函数。

def f(varr):
varr = varr * 2
varr += varr
#…….
return varr

def f1():
pass

  1. 假设你正在编辑一个python源程序，想要生成一个大纲，基本思路是找到所有函数头部的第一行，将它们复制到文件结尾，比如第2题的例子中会生成如下大纲,(提示使用global命令)
    def f(varr):
    def f1():

  2. 考虑如下步骤：
    a. 将第2题中的python源程序示例中的行首先高亮选择，接下来给出相应的命令以将其保存到testw9.py中
    b. 水平开启一个新的窗口编辑testw9.py文件
    c. 现在在新的窗口testw9.py中，处于正常模式，请给出相应的命令高亮选择所有的行
    d. 接下来在c的基础上，对于所有行都要在前面加上行号，请给出相应的命令。(提示使用外部命令cat的加上行号的功能)

  3. 请大家自己准备若干文本文件，然后使用多窗口的方式，开启多个窗口同时编辑多个文件，熟悉如何使用:split :vsplit以及相应快捷键s v开启多个窗口，使用:close和快捷键c关闭窗口，使用:edit命令在当前窗口编辑其他文件，使用:ls命令查看缓冲区列表，使用:b 部分文件名 编辑已在缓冲区列表中的缓冲区。使用w、hjkl在窗口之间切换

  4. 我们知道如果文本文件中的一行比较长的时候，且设置wrap选项时会多行显示，这样原来的jk就不太方便了，只能移动到下一个和上一个物理行，显然使用gj和gk命令移动到下一个和上一个显示行更加符合我们的习惯，请使用相应的map命令将j映射为gj，将k映射为gk。

  1. 下面是有关用户管理的内容，要求使用useradd/usermod/groupadd等命令完成
    a. 增加一个普通用户tony，自动创建用户主目录，并设置shell为/bin/bash
    b. 增加一个用户组 linux
    c. tony要从普通用户变为管理员用户，同时tony也要加入到linux组中

我的答案

  1. ‘<,’> norm a.
  2. $}:s/varr/var/g
  3. g/^def\s+/copy $
  4. a. g/.*/
    :.,.+7w testw9.py
    b. :vsplit testw9.py
    c. $G
    d. :’<,’> !cat -n
  5. noremap j gj
    noremap k gk
  6. a. sudo useradd -m -s /bin/bash tony
    b. sudo addgroup linux
    c. sudo usermod -a -G sudo,linux tony

参考答案

  1. ‘<,’>normal a.
    或 ‘<,’>normal f i. 或’<,’>normal t a.
    第2种写法考虑了多位数字的情况，找到第一个空格，然后插入。

  2. .,/def /-1 s/varr/var/g
    或者
    .,/def\s/-1 s/varr/var/g
    从当前行开始，接下来找到def那一行，然后-1，表示那一行的前一行

  3. :%g/def /copy $
    或:1,$/def /copy $

  4. ‘<,’>w testw9.py
    :sp testw9.py 也可以:new testw9.py 尽管:new一般用于新建一个文件
    ggVG
    :’<,’> !cat -n
    注意:!cat -n 与 :% !cat -n的区别，前者表示执行外部命令，然后返回vi。后者是指定范围内的行交给cat -n处理，然后替换指定范围内的行

有同学采用如下实现，
:’<,’> g/^/ s//=line(‘.’).’ ‘/
用到了课堂上没有介绍的知识，我简单解释一下
g/^/
匹配行首，这样每一行都匹配，对于每一行执行:s命令，s//省略了pattern，相当于上一个pattern即^,也就是行首进行替换,后面用到了vim脚本语言中的表达式的概念, =line(‘.’) 表示这里替换为表达式line(‘.’)的内容，即返回行号，然后line函数之后的.表示的是与后面的包含空格的字符串拼接在一起，综合下来s命令就是行首增加了行号加上空格。

  1. nnoremap j gj
    nnoremap k gk

  2. useradd -m -s /bin/bash tony
    groupadd linux
    usermod -a -G sudo,linux tony
    注意应该是加入到两个组中，主组并不需要更改。同时加入两个group时，之间以逗号隔开
