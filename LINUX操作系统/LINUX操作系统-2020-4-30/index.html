LINUX操作系统--第十节(连接和权限)

发表于 2020-04-24

题目

  1. 要以另外一个用户test身份启动一个全新的登录shell，请给出相应的sudo命令。

  2. 给出一个普通文件的文件名，你能够知道该文件是硬连接吗？也就是你能够知道其是否是使用ln命令创建的吗？用一句话说明理由。

  3. 请创建一个名为slnk的指向linux/slides/1.pdf的符号链接，给出相应的命令。接下来cat slnk，结果报错
    cat: slnk: No such file or directory
    请问是什么原因？

  4. 假设某个目录的权限如下，并且假设slides目录下有一个名为readme的文件，用户test有readme的读权限，
    d–x–x— 3 test pubs 512 2020-03-10 15:16 slides
    当前用户为test，请问：
    (a) cd slides会出错吗？请说明理由
    (b) ls slides会出错吗？请说明理由
    (c) cat slides/readme会出错吗？请说明理由

  5. 把下列符号方式表示的权限转换为八进制数字形式：
    (a) rwxr-xrw-
    (b) –x-w-r–

  6. 你希望新建的普通文件的权限为 rw-rw—-，应该怎么设置？

我的答案

  1. sudo - test
  2. 不可以，因为硬链接产生的文件名和原本的文件名是独立的，只是对应的inode相同
  3. 应该是因为1.pdf被删除了或者位置被移动了
  4. (a) 不会，因为test是该目录的拥有者，有x的权限
    (b) 会，因为test在该目录上并没有r的权限
    (c) 不会，因为test在目录上有x权限，可以找到readme文件；并且test有readme的r权限
  5. (a) 756
    (b) 124
  6. umask 117

参考答案

  1. 要登录shell，如果sudo test，表示执行test命令，因此需要-u选项传递
    sudo -i -u test
    另外也可以sudo和su命令结合
    sudo su - 或sudo su -l

无法知道是否是使用ln命令创建的硬连接。因为创建硬连接不过是给某个文件添加了另外一个名字而已，无法区分哪个名字在前哪个名字在后。

  1. 符号链接缺省会展开的，因此错误原因是找不到文件和目录，这样有可能是linux/slides/1.pdf不存在，或者途中的目录不存在。

  2. (a) 无错。当前用户test与slides目录的拥有者一致，只需要查看第一组权限。有x的权限，因此允许切换到slides目录
    (b) 有错。ls slides要浏览目录需要有r权限，当然x权限是首要的
    (c）无错。slides有x的权限，从而可以找到readme文件，而readme有读的权限，因此可以查看。

  3. (a) 756
    (b) 124

  4. 其他用户没有rw的权限，考虑到目录一般会允许有x的权限，umask 006
