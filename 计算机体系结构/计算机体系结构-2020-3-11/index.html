计算机体系结构--第二节(指令集体系结构)

发表于 2020-03-11

Review

  1. 应用需求推动了技术的发展
  2. 软件解决的是不同平台上的兼容性问题
  3. Stack Architecture
    • ALGOL使用栈结构简化编译
    • 简化了表达式计算、递归调用、中断处理、…
  4. IBM360: GPR(通用寄存器)结构
  5. 随着时间发展，指令集越来越复杂，越来越依赖于寄存器
  6. IBM360设计了指令集的结构，隐藏了技术实现的细节
    • 是一个真正的指令集，实现了硬件和软件之间的接口
  7. 指令集处于上层的应用软件和下层的应用之间，为上层的应用软件访问下层的硬件和执行各种功能提供了接口

指令集设计的原则

  1. IBM360体系结构的成功说明了指令集的设计对于计算机的发展具有重要的作用
  2. 设计计算机指令集体系结构时，从以下五个方面考虑：
    • 数据存储
    • 内存地址的访问模式
    • 指令集的计算
    • 指令集的格式
    • 指令集的编码

存储架构：定义

  1. 现在所有的计算机体系架构都遵从冯·诺依曼架构，即所有的数据和指令都存在计算机的主存中
  2. 对于处理器，存储体系结构涉及：
    • 操作数存在什么地方，方便执行运算？
    • 执行运算后的结构存在哪里？
    • 怎么规范操作数？

存储架构：普遍设计

  1. Stack Architecture(栈的体系结构)
    • 操作数都隐式地存在栈的顶部
  2. Accumulator Architecture(累加器的体系结构)
    • 一个操作数可以隐式地存在累加器中(以一个特殊的寄存器的形式出现)
  3. General-purpose register architecture(通用寄存器的体系结构)
    • 只有显式的操作数
    • Register-memory architecture
      • 一个操作数在内存
    • Register-register architecture
      • 需要把内存中的数据load到寄存器，寄存器中的数据store到内存
    • RISC计算机通常使用 Register-Register 设计
    • CISC计算机通常使用两种混合的设计
  4. Memory-memory architecture(存储器-存储器架构)
    • 所有操作数都在内存中
  5. 不同的存储结构对程序会有比较大的影响
  6. 【疑问】：此处的显式和隐式是什么意思？

内存地址和内容

  1. k-bit的地址，地址空间为2k
  2. 每一次内存传送一个字(n-bit)
  3. 在现有的计算机中，内存地址的单位是1-byte
  4. 处理器访问数据时，通常支持多个字节的字，对于多个字节
    • 大端存储：权重最大的字节保存在低地址
      • 0xDE AD BE EF，按地址从小到大的顺序：DE AD BE EF
    • 小端存储：权重最大的字节存在高地址
      • 0xDE AD BE EF，按地址从小到大的顺序：EF BE AD DE

寻址模式

  1. 寻址就是：操作数在汇编语言/机器语言/指令中存储的位置
  2. MIPS架构中，有三种不同的寻址模式
    • 计算器
    • 立即数
    • 偏移模式
      • 操作数在内存中，通过基地址和偏移地址获得

    • 程序对指令的使用具有鲜明的特点：优化前四种指令

指令长度

  1. 变长指令集：
    • x86：1-17-byte long
    • 超过了处理器的数据宽度，需要多次取值和译码
    • 指令集的描述更加灵活，也更加复杂，更加紧致的指令集
  2. 定长指令集：
    • 用于大部分RISC架构(Reduced Instruction Set Computers)
    • MIPS：4-byte
    • 取值和译码更容易
    • 可以获得更高的主频：实现流水线和并行更加容易
    • 指令位数固定，指令较少
  3. 混合指令集：
    • 变长指令和定长指令
    • 扩展指令使用变长指令架构

指令的构成

  1. 一条指令通常包括：
    • 操作码：唯一的代码，表明指令执行的操作
    • 操作数：0..*多个
  2. 运算中包括了数据类型和数据宽度，典型的数据类型和长度：
    • Character (8 bits)
    • half-word (eg: 16 bits)
    • word (eg: 32 bits)
    • single-precision floating point (eg: 1 word)
    • double-precision floating point (eg: 2 words)
  3. 任何新的32位体系结构的期望：
    • 支持8位，16位和32位整数以及32位和64位浮点运算
    • 64位体系结构也需要支持64位整数运算

指令编码

  1. 指令如何用二进制的形式表示，方便处理器执行
    • 涉及的问题：指令的长度、执行的速度、设计的复杂度
  2. 可以由以下因素来决定：
    • 寄存器的数目
    • 寻址的模式的种类
    • 指令中操作数的个数
  3. 不同的竞争力量：
    • 有许多寄存器和寻址模式
    • 减少代码大小
    • 指令长度易于处理（定长指令易于处理）
  4. 编码选择：变长、定长和混合编码
  5. 【疑问】在同一个体系结构中，如果指令本身是定长的，那么对应的编码也是定长的吗？存在对应关系吗？

定长指令：编码

  1. 指令的长度固定，带来了更多的挑战：
    • 如何在相同的指令长度中放入众多类型指令？
    • 先考虑限制条件较多的指令
    • 比如A：2 operands，each 5-bit； B：1 operand of 5-bit；A的限制条件更多
  2. 扩展操作码(Expanding Opcode)模式：
    • 虽然指令长度是相同的，但是操作码，也就是指令类型长度是可变的
    • 提高指令位数的利用率
  3. 尝试一：固定操作码的长度
    • 一些指令有bit浪费
    • 支持的指令数受操作码长度限制
  4. 尝试二：扩展数的模式
    • 支持更大的指令集
    • 问题：因为操作码长度不同，那么如何区分不同操作码长度的指令类型？
      • 前缀码的方式
    • 最多支持多少种不同的指令？

CISC vc RISC：

  1. 复杂指令集计算机(CISC，Complex Instruction Set Computer)
    • 示例：x86-32（IA32）
    • 单条指令执行复杂的操作
      • VAX体系结构有一条指令可乘多项式
    • 代码紧凑，对内存的使用率低
    • 实现复杂，没有硬件优化的空间
      • 早期CISC架构的处理器主频较慢
  2. 精简指令集计算机(RISC，Reduced Instruction Set Computer)
    • 示例：MIPS，ARM
    • 使指令集小而简单，使构建/优化硬件更加容易
    • 负担从硬件转移到软件，结合更简单的操作来实现高级语言语句
    • RISC的指令集很少，每个指令能做的事情又很简单，导致送入CPU后执行时间都很相近，总线不太常塞车。RISC的CPU就可以用较低时延、较小尺寸，来完成CISC差不多的任务。
      • 较低时延可以让散热减少，功耗降低
      • 较小尺寸可以让做出小巧的嵌入式产品机会变大

RISC-V指令集

  1. 贯穿教材，作为样例，用于教学
  2. 具有很多现代很多指令集的有点
  3. RISC芯片通常用于嵌入设备
    • 基于RISC-V的芯片被应用到消费者电子产品，网络/存储的设备，照相机，打印机，…

算数运算

  1. 加减，三个操作数
    • 两个源操作数，一个目的操作数
    • add a, b, c // a = b + c
  2. 在RISC-V中，所有的算数运算都是这种形式
  3. 设计原则1：Simplicity favours regularity(规则简化设计)
    • 规律性使实施更简单
    • 简单性以更低的成本实现更高的性能

寄存器操作数

  1. 在RISC-V指令集中，所有算数运算，都使用寄存器作为操作数
  2. RISC-V有32 × 64-bit的寄存器
    • 编号：x0 -> x31
    • 32-bit -> 字
    • 64-bit -> 双字
  3. 为什么使用32个寄存器(不用8个或者64个)？
    • x86中只有8个通用寄存器，当数据较多时，必须通过栈来传递，麻烦
    • 设计原则2：Smaller is faster(越小越快)
      • 电路越复杂，时钟信号越难控制
      • 寄存器 vs 主存的访问速度

RISC-V 的寄存器

  1. 为了方便编译器生成程序，我们对寄存器做了规范
  2. 这个规范只是为了让大家遵循，不一定强求：
寄存器 存储内容
x0 the constant value 0，不能被修改
x1 return address
x2 stack pointer
x3 global pointer
x4 thread pointer
x5 – x7, x28 – x31 temporaries，存放变量，存放中间结果
x8 frame pointer，栈的大小
x9, x18 – x27 saved registers
x10 – x11 function arguments/results
x12 – x17 function arguments

内存操作数

  1. 通常而言，数据都会放在内存中
  2. 内存通常用于存储复杂的数据结构：
    • 数组、结构、动态的数据
  3. 我们的处理器只支持简单的数据类型的算术运算，所以
    • 把数据从内存加载到寄存器
    • 计算完成之后把数据结汇内存中
  4. 内存按照字节为单位来存放数据
    • 每一个地址标识8-bit的数据
  5. RISC-V是小端存储
    • 权重小的byte在低地址
  6. 不同于其他RISC的指令集，RISC-V不要求word在内存中对齐
    • 跟x86具有相同的特性

寄存器 vs 内存

  1. 寄存器的访问速度较快
    • RISC-V所有的算术运算都在寄存器中
  2. 对内存数据的操作需要load和store
    • 需要执行更多的指令
  3. 编译器尽量把变量放在寄存器中
    • 把不常用的数据放在内存中
    • 寄存器的优化是十分重要的！

立即数

  1. 指令中指定常数数据
  2. 快速处理常见问题
    • 通常常数都很小
    • 取指时把数据读到处理器中，避免从内存中load

符号扩展

  1. 用更多位表示数字
    • 保持数值不变
  2. 把符号位复制到高位上：
    • 无符号数，高位补0
    • 有符号数，符号位是什么，高位补什么
  3. 符号扩展不是隐式的(不是自动完成的)，在RISC-V中有相应的指令需要被调用：
    • lb：有符号扩展
    • lbu：无符号扩展

表示指令

  1. 指令以二进制来进行编码
    • 称为机器码
  2. RISC-V指令被编码为32-bit的字
    • 少数几种格式，包含：操作码，寄存器，…
    • 规律性编码，方便硬件实现

  4. 立即数出现在算数运算和load指令(偏移地址)中，通常是有符号的数

    • 好的设计需要好的折中
    • 格式越多，解码越复杂，但统一允许32位指令
    • 保持格式尽可能相似

条件操作

  1. 如果条件为true，跳转；否则，继续执行
  2. beq rs1, rs2, L1
    • 如果 rs1 == rs2， 那么跳转到L1
  3. bne rs1, rs2, L1
    • 如果 rs1 != rs2，那么跳转到L1
  4. beq x0, x0, Exit
    • Exit: ...
  5. blt rs1, rs2, L1
    • rs1 < rs2 –> L1
  6. bge rs1, rs2, L1
    • rs1 >= rs2 –> L1

有符号数和无符号数比较

  1. 有符号比较：blt, bge
  2. 无符号比较：bltu, bgeu

函数调用

  1. 步骤：
    • 传参：把变量放在寄存器 x10-x17
      • 超过8个参数放在栈(内存)上
    • 把程序执行的控制权移交给被调用的函数
    • 为被调用函数分配栈空间
    • 执行运算
    • 把返回值放在寄存器中
    • 返回到调用的地址(返回地址在寄存器 x1 中)，即把控制权移交给调用者

函数调用的指令

  1. 函数调用：jump and link (jal)

    jal x1, ProceduralLabel
    • 返回地址放入x1
    • 跳转到ProceduralLabel

  2. 函数返回：jump and link register (jalr)_

    jalr x0, 0(x1)
    • 类似于jal，但是跳转到 0(x1)
    • 用x0作为目标寄存器，不可修改
    • 可以用来计算跳转

  3. 被callee使用的寄存器，要考虑先将寄存器的数据存在栈中，返回时从栈中恢复

  4. 当调用其他函数时，需要考虑：

    • 返回地址被存放在 x1 寄存器中，这个寄存器的内容会被覆盖，需要存放在栈中
    • 一些参数和临时变量也需要存放在栈中，这些寄存器是否保留由调用者决定

内存布局

  1. Text：程序代码
  2. 静态数据：全局变量
    • x3(global pointer)初始化为地址，允许在此段中有±偏移
  3. 动态数据：堆
  4. 栈：自动存储
    • 变量、数组

栈上的局部数据

  1. 局部数据被callee分配

    • C：自动变量

  2. 函数帧，栈帧(活动记录，activation record)

    • 编译器用来管理栈的存储

    • FP不变，SP指向栈顶

Byte/Halfword/Word 操作

  1. load时需要指明长度和有符号数还是无符号数
  2. Load byte/halfword/word: Sign extend to 64 bits in rd
    • lb rd, offset(rs1)
    • lh rd, offset(rs1)
    • lw rd, offset(rs1)
  3. Load byte/halfword/word unsigned: Zero extend to 64 bits in rd
    • lbu rd, offset(rs1)
    • lhu rd, offset(rs1)
    • lwu rd, offset(rs1)
  4. Store byte/halfword/word: Store rightmost 8/16/32 bits
    • sb rs2, offset(rs1)
    • sh rs2, offset(rs1)
    • sw rs2, offset(rs1)

32-bit常数

  1. 在I-format中，立即数只有12-bit，但是在否写情况下可能需要32-bit的常数
  2. lui rd, constant：
    • 拷贝高20-bit到rd的[31:12]
    • 有符号扩展至[63:32]
    • 将rd的[11:0]置为0
  3. 将32-bit常数0x00 3D 05 00放进寄存器中：
    • lui x19, 0x003D0
    • addi x19, x19, 0x500

分支寻址

  1. 分支指令指定

    • 操作码，两个寄存器，目标地址

  2. 大多数分支目标都在分支附近

    • 前进或后退

  3. SB-format：

    • 方便硬件设置

  4. PC-relative 寻址

    • 地址是64-bit，但是立即数只有12-bit，所以都会采用这种寻址方式
    • Target address = PC + immediate × 2 (必须两个字节)

Synchronization(同步化)

  1. 两个处理器共享同一片内存空间
    • P1写，P2读
    • 如果没有同步化机制，那么会产生race，访问结果依赖于访问顺序
  2. 实现同步，需要硬件支持
    • 对内存操作的原子读写
    • 在读写中不会有其他访问
  3. 可以用一条指令实现：
    • eg：实现在内存和寄存器间的原子性交换
    • 或原子性的一对指令

RISC-V 提供的同步化

  1. Load reserved：lr.d rd, (rs1)
    • 将内存地址rs1中的值load到rd寄存器
    • 把这个地址保护起来
  2. Store conditional：sc.d rd, (rs1), rs2
    • 将rs2中的值存到rs1中
    • 成功：location从load后没有改变，返回0(rd中)
    • 失败：location改变了，返回 non-zero(rd中)
