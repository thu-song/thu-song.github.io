操作系统Ⅰ--第二节

发表于 2020-03-11

多处理器系统

有的计算机系统只包含单一的CPU
有的包含不同类型的CPU
如果把每一个CPU都看成一个系统
所以称为tightly-coupled系统

提高了吞吐量
经济资源
提高了可靠性：多个核，其中一个核坏了

多个处理器可以有相同类型：
如果类型不一致，非对称
如果类型一样，也可能是非对称的

对称指的是，一旦启动，多个CPU是一样的，不加区分

bootstrap 只能在一个处理器上运行
在上电时，只有一个处理器被启动

冯诺依曼结构

CPU以cache作为中介，以透明的方式访问memory，获得指令和数据

在CPU和设备之间：
- CPU可以直接对设备进行读写：IO request
- 设备本身不会做任何事情，必须由CPU产生IO request
- 设备做某些事情，围绕data
- 设备可以产生中断

对称多处理器

  1. 每个CPU有独立的register和cache

多核

  1. 单一的芯片，内部有多个核(重复的CPU功能件)

  2. UMA 和 NUMA

    • 单处理器中不存在这个概念
    • UMA，对于所有内存和处理器，访问速度都是一样的
    • NUMA，有的处理器访问一些内存快，有些慢
    • NUMA代表非对称的，可以做优化，使处理器尽可能访问离他最近的内存

  3. 每一个核内部也有自己的cache

    • 同样芯片也可以有cache，不矛盾

簇系统

  1. 操作系统的阵列
    • 所有计算机由特定的网络连接起来的系统中，服务于共同的目的
    • 不是通过总线连接，而是通过网络连接
  2. SAN，在一个网络中共享存储
  3. 满足于HPC，高性能计算，
    • 协调CPU，实现高效并行
    • 多个处理器，做大型计算
  4. DLM
    • 事务性的，分布式锁管理器，在多个系统中管理
  5. 计算机和计算机通过高速的interconnect互联，高速的交互通道

操作系统本身的结构

  1. 一个用户的程序往往没办法高效利用计算机资源
  2. 在同一个计算机上运行多个程序——multiprogramming
    • 任务的子集放在内存中
    • job scheduling在内存中选一个任务运行
      • 一旦任务需要等待或者空闲，CPU把任务拿出来，切换到另一个任务
  3. 切换的方式
    • 时分，CPU定时拿出任务，切换任务
      • 优点：响应性好
        • 对系统做出指令动作到指令做出第一个可观察的动作所需的时间
        • 有可能进程需要访问的数据不在内存中，如果数据不在主存中，那么不会被调度
        • 如果进程中需要的数据不能完全放入内存中，CPU提供了虚存(大于物理内存大小)
        • 虚存中数据够了，CPU也可以切换到任务
          • 如果运行中使用的数据不在内存中，由虚存解决，而不是CPU把任务停下来

对进程做一些限定
- 在投入运行之前，可运行的进程的数量是可知的、不变的
- 在运行前加载到内存中
- 如果每次加载的初始地址是固定的，那么就是单进程的系统，DOS系统

软中断可以主动产生也可以被动产生

  1. 如果一个进程进入了死循环，OS如何处理

  2. OS必须对所有进程持怀疑态度

  3. OS必须可以对进程进行管理

  4. 在硬件上提供一种机制，保护自身和其他系统组件，保护进程不会被破坏

    • 双模式：用户态和kernel态
    • 两种模式是硬件的状态，是在CPU中的状态，而不是主板上的一个开关 —— mode bit
      • 区分CPU处于不同的模式
      • 将某些指令设计为privileged，某些指令在mode bit设定为特定值时，才会表现为特定行为
      • 两种方式处理模式：
        • 特权指令，在kernel上正常运行，用户态不会产生可观察到的结果，会产生异常
        • 私有指令：CPU早期，在kernel上以某种方式运行，在用户态不会产生异常，但是会产生不同的行为
      • popf可以更改mode bit：可以随意更改
        • 正常是允许kernel转换为user
        • 所以重新定义了popf的行为，user模式下，无论当前栈中内容如何，在覆盖时，保持mode bit不变

  5. 虚拟机需要区分：kernel态，用户态，虚拟机上的kernel态

从用户态切换到kernel态

  1. 异常和陷阱的中断服务程序入口在kernel态
  2. 特殊的中断——系统调用入口
  3. 完成或者无法完成，必须返回到用户态
    • jump不能从kernel转换到用户
    • 指令自己把kenerl转换到用户，然后jump
    • popf方式可以转换，中断产生，中断返回
      • 把中断返回地址压到栈中，返回
