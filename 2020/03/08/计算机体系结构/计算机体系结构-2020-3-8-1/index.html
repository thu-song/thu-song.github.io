计算机体系结构--第一节(History)

发表于 2020-03-08

History Concepts

计算机体系结构：历史

  1. 我们为什么关心旧的概念、观点？
    • 解决问题的过程是逐渐演化的过程，通过旧的设计过程，这些决定，计算机的发展是工程和科学混合的过程，通过历史的变化了解如何做决策
    • 早期做决策是因为技术的限制，现有的技术在将来也会面临同样的问题
    • 如果我们不从历史吸取教训的话，将来也会重复错误
      • 早期时候我们设计的主要是大型机，我们设计大型机中遇到的问题在微型机中也会遇到相同的问题
      • 比如虚拟机的硬件支持在大型机中已经有了，但是好长时间之后才应用到微型机中

模拟(analog)计算机和数字计算机

模拟(analog)计算机

  1. 模拟计算机把问题变量用物理的数字表示(比如位移、电压)，用等比的物理行为表示计算结构
  2. 当物理规律复杂的时候，模拟计算机更为方便，比如模拟飞机飞行时的气流变化，用等比的飞机模型在风洞中做测试

数字计算机

  1. 数字计算机把问题的变量用一些离散的数字表示
    • 离散化可以排除噪音，噪声引入并不影响值的精度
    • 但对模拟计算机而言，连续发生的变化，受噪音影响
  2. 让精确的计算和可重复的计算成为可能
    • 相同的输入总是得到相同的输出
  3. 不再依赖于物理规律，受限更少

Charles Babbage(1791-1871)

  1. 英国数学家，剑桥大学，对许多方面都感兴趣
  2. 提出计算机的概念：在欧洲的中世纪，商业发达，商业中存在很多计算，对于复杂的计算，会有类似对数的表，但是他发现表中存在错误，所以他就想制造机器来计算和打印正确的表格
  3. 受到了早期人力“计算机”的启发

Difference Engine 1822

  1. 差分计算机：任意连续的函数都可以用多项式逼近，多项式可以用差分表计算
  2. n阶差分(常数，与n无关)，计算任意的n

实现差分机器

  1. 手摇的机械计算器，采用十进制
  2. Babbage没有完成DE，设想了一般的分析机器
  3. 瑞典的Schuetz在1855年实现了DE，卖给了英格兰政府
  4. 现代仿制了DE2，包括了打印功能

Analytical Engine 1837

  1. 第一种通用的数字计算机
    • 经过了多次迭代的设计
  2. 具有现代计算机中很多主要的部件
    • Store(存储)：存数字和中间结果，(1000 digital words, 40-digits each)
    • Mill(磨坊)：用来计算加乘除的运算旦苑
    • 支持条件跳转和循环，当计算发生溢出可以抛出异常
    • 形成了microcode(硬件指令和程序员指令的编译器)，叫做”Barrel”
  3. 通过打孔的卡片编程、输入输出
  4. 指令的卡片中有操作码和操作数的地址，3D的格式，两个source和一个destination，都在store中
  5. 通过机械的改变卡片的顺序实现条件跳转
  6. 只生成的一部分部件

分析机的设计决策

  1. 十进制，通过机械的齿轮方式实现
    • Babbage考虑过二进制等，但是这些和十进制相比没有特别的优势
  2. 40位的精度(相当于 > 133 bits)
    • 通过放大位数减小处理浮点数的误差(向上取整和向下取整)
  3. 用”锁(locking)”或者机械放大(amplification)的方式克服计算中的噪声
    • 比如噪声中导致的位移
    • 类似数字计算机中电路中非线性的收入(non-linear gain)
  4. 快速的进位
    • 这样的方式是现在CMOS实现加法器的机械般

Ada Lovelace (1815-1852)

  1. 建立了循环和子程序的概念，早期接触计算机是从意大利人写的Baddage讲义，之后产生兴趣
  2. 参考笔记，试图用分析机，计算贝努力数在，此过程中画的流程图就是软件设计流程图最早的形式
    • 第一个程序
  3. 甚至用计算机做其他更多的事情，不仅是计算表格
  4. 英国数学家，对如何对大脑建模有很大的兴趣，程序设计流程图
  5. 为计算机写程序的第一人

早期的可编程计算器

  1. 20世纪上半叶，数字计算机价格昂贵，模拟计算机流行
    • 可以计算炮弹的弹道
  2. 30年代后期和40年代已经有了可编程的计算机
    • Atansoff Linear Equation Sovler (1939)
    • Zuse Z3 (1941)
    • Harvard Mark Ⅰ (1944)
    • ENIAC (1946)

Atansoff Linear Equation Sovler (1939)

  1. 线性求解器，固定功能的求解器，最多可以计算19个线性方程
  2. 二进制计算(50-bit 不只是浮点数)
  3. 动态存储(电容存储)
  4. 计算用真空管
  5. 1973年作者被认为是自动电子数字计算机的发明者

Zuse Z3 (1941)

  1. 设计者是Konrad Zuse在二战间用2000个中继器做成
  2. 支持浮点运算，并且具有Exceptional Values(+/- infinity，undefined)
    • 1-bit符号，7-bit 指数，14-bit数值
  3. 64个字的存储能力
  4. 两阶段的流水线：1)取指和执行； 2)写回
  5. 没有条件分支
  6. 通过纸袋编程

Harvard Mark Ⅰ (1944)

  1. 由哈弗大学的Hoeard Aiken提出，IBM资助和建造
  2. 大部分是机械的设备，一些电子控制的中继器和控制杆
  3. 75万个部件
  4. 只能存储72个数字，每个23位，十进制
  5. 加法：0.3s，乘法：6s，除法：15s，三角函数：>1min
  6. 指令在纸带上，是两地址的表达格式(类似×86的表达)
  7. 可以长时间自动运行
  8. 循环是用胶水粘成环形
  9. 不需要条件分支，只需要粘成环形
  10. 虽然说在设计方案中提到Babbage，但是功能少于分析机

ENIAC (1946)

  1. 第一台通用的电子计算机
  2. 在二战期间在Upenn Moore大学秘密开始，为美国陆军计算火炮设计射击表
  3. 17468个真空管组成
  4. 12个十进制的累加器
  5. 有条件分支
  6. 通过电路板的插入和开关编程控制，非常消耗时间，运行的时间可能远远小于编程的时间
    • 对机器编程大概需要1天的时间
  7. 完全用电子指令控制，包括取指和执行
    • 10位的10机制乘法用2.8ms
  8. 执行完全受限于输入输出
  9. 因为使用了大量的真空管，所以很容易坏掉，最长的使用时间是15天

EDVAC(Electronic Discrete Variable Automatic Computer， 离散变量自动电子计算机)

  1. 是一台美国早期电子计算机。与它的前任ENIAC不同，EDVAC采用二进制，而且是一台冯·诺伊曼结构的计算机。
  2. ENIAC开始让我们讨论如果把程序存在计算机中，可以加速编程也会让运行简单
  3. 冯·诺依曼在Upenn和一些人讨论，写了”First Draft of a report on EDVAC”
  4. 会议的参与者之一Herman Goldstine，起草了草案在很多机构发表，激发了大家研究存储编程(stored-peogram)的基础
    • 这个思想是很多人一起讨论出来的，但是大家把它归功于冯·诺依曼
    • Maurice Wilkes受到报告的鼓舞，决定到美国工作室造计算机
  5. 后来在1948年，ENIAC的修改版本可以运行存储的程序，但是比原来慢了6倍
    • 因为IO的限制，这种速度下降其实并不重要，从实际的实用角度上提高了生产率
  6. 1951年EDVAC被构造成功，并可以使用
    • 因大学的纠纷而延迟

Manchester SSEM “Baby” (1948)

  1. Manchester 大学先规模的实验，主要是用阴极射线管(CRT)做计算机的存储，取代水印征集线
  2. Williams-Kilburn Tubes是第一个支持随机访问的电子存储设备
  3. 32个字，一个字32-bit，支持累加，有PC
  4. 在1948年6月份运行了世界上第一个存储-程序
  5. 之后形成了 Manchester Mark-1 全尺寸机器
    • 引入了 index 寄存器
    • 被 Ferranti 商业化

Cambridge EDSAC (1949)

  1. Maurice Wilkes从美国工作室回来在剑桥造了一台运行存储-程序的计算机
  2. 采用汞延迟线做存储，做多可以存储 1024 个字，17-bit(1-bit做延迟线的边界padding)
  3. 二进制补码运算
  4. 累加器有指令集，带有用于索引的自修改代码
  5. David Wheeler，世界上第一个有计算机博士学位的人，发明了这台机器的子程序(subroutine，”Wheeler jump”)
    • 用户构建了一个有用的子例程的大型库
  6. 英国的第一个商用的计算机leo-Ⅰ就是基于EDSAC，在1951年跑商业软件
    • 用于LEO的软件仍在1980年代在ICL大型机上进行仿真运行！
  7. EDSAC-Ⅱ是第一台具有微控制器单元的机器
