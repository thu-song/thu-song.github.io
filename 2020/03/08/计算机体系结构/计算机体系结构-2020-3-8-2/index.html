计算机体系结构--第一节(History Modern)

发表于 2020-03-08

Commercial computers：BINAC (1949) and UNIVAC (1951)

  1. Eckert 和 Mauchly 与U.Penn发生专利纠纷之后离开了，成立了Eckert-Mauchly计算公司
  2. 世界上第一台商用计算机为BINAC，有两个处理器，互相check，为了正确性
    • BINAC移交给用户之后，几乎没有正常的工作过
  3. 第二台商业计算机是 UNIVAC
    • 用汞延迟线作为存储，可以存1000个字，每个字可以存一个字母
    • 1952年预测了总统的选举
    • 经常被误认为是IBM UNIVAC

IBM 701 (1952)

  1. 第一台商用的精确计算的计算机
  2. 内存使用了72个William’s真空管，每个可以存1Kib，一共可以存2048个字，每个字是36-bit
    • 每个内存周期是12μs
  3. 累加器ISA带有乘法和商的寄存器
  4. 18-bit和36-bit的数字是有符号位的定点数
  5. 谬论 —— Thomas Watson Sr/Jr说：计算机的市场只需要五台计算机
  6. 实际上TWJr在股东大会上说，在卖机器的路上期望能拿到五台机器的订单，回到IBM时拿到了18台机器的订单

IBM 650 (1953)

  1. 第一台大规模生产的计算机
  2. 相比701是较低端的系统，使用磁鼓类的存储，数字串行的ALU
  3. 大概生产了2000多个
  4. 虽然是十进制的计算，但是每次只做一位的计算，串行计算

IBM 650的指令集

  1. 地址和数据都是用10位十进制的字
  2. 指令编码：
    • 2位操作码，基本指令集有44条，加上选择之后，可以表示97条指令
    • 4位的数据地址
    • 4位的下条指令的地址
      • 程序员可以调整代码减少延迟
  3. 添加了与track上进行值比较的特殊指令

早期的指令集

  1. 大部分都是单地址的累加器，高速的电路昂贵
    • 基于早期的”计算器”模型
  2. 随着时间发展，软件的需求决定了ISA
  3. 添加了索引寄存器（Kilburn，Mark-1），以避免需要自我修改的代码来遍历数组
  4. 随着时间发展，更多的索引寄存器被引入
  5. 更多的操作基于寄存器
  6. 最终形成了通用寄存器的机器和需多相关的指令

Burrough’s B5000 Stack Architecture: Robert Barton, 1960

  1. 基于栈的计算机
  2. 使用高级语言（ALGOL）完全向程序员隐藏指令集
  3. 使用堆栈体系结构简化编译、表达式求值、递归子例程调用、中断处理等

表达式求值

  1. 逆波兰式，后续表达式

IBM’s Big Bet: 360 Architecture

  1. 在1960s之前，IBM由读个不兼容的计算机组成：
    • 701 -> 7094
    • 650 -> 7074
    • 702 -> 7080
    • 1401 -> 7010
  2. 每一个系统都有自己的：
    • 指令集
    • IO存储也不同(磁带、磁鼓、磁盘)
    • 编译器、汇编器和库…
    • 商业市场也不同(商业、科学计算、实时计算…)

IBM 360 : 设计的前提 —— Amdahl, Blaauw and Brooks, 1964

  1. 设计本身要有一定的成长空间，后续的机器可以通用
  2. 连接IO设备的通用方式
  3. 关注整体性能，每个月求解问题的个数 —— 从编程的角度来看
  4. 机器自身要有监管能力 —— 不需要人工的干预
  5. 机器硬件要内置错误的检测功能，要能够定位，减少机器宕机的时间
  6. 具有冗余的IO设备，组装系统要简单，提高容错性
  7. 一些问题需要多于36位的浮点数

栈结构 vs 通用寄存器结构 —— Amdahl, Blaauw and Brooks, 1964

  1. 下推堆栈组织的性能优势来自快速寄存器的存在，而不是它的这种使用方式。
  2. 在栈结构中，数据总在栈的上方，它的收益大概是50%，因为常量和通用子表达式的方式，使用寄存器更为合适
    • 因为子表达式要反复计算，寄存器可以表达
  3. 由于隐含地址而导致的指令密度优势可以通过允许短地址指定寄存器的方式解决 (?)
  4. 有限深度堆栈的管理会导致复杂性
  5. 递归子例程的优势只有借助独立的寻址，与堆栈无关的方式才能实现
  6. 将可变长度字段适合到固定宽度的单词很麻烦

IBM 360: 通用指令集(A General-Purpose Register (GPR) Machine)

  1. 处理器状态
    • 16个 32-bit 的通用寄存器
      • 既可以用作index，也可以用作base
      • 第0个寄存器有特殊的属性
    • 4个 64-bit 的浮点寄存器
    • 一个表示程序状态的字(PSW，Program Status Word)
      • PC，状态码和条件码
  2. 一个32-bit机器，有24-bit的地址
    • 没有指令包含 24-bit 地址
  3. 数据格式：8-bit字节，16-bit半字，32-bit字，64-bit双字
    • IBM360的出现决定了现在为什么 1-byte = 8-bit

IBM 360：最初的实现

  1. IBM 360隐藏了不同模型间技术的差异
  2. 里程碑：第一个真正将ISA设计为便携式硬件-软件接口的机器
  3. IBM 360的结构经过小的改动直至今天还在使用

总结

  1. 计算机体系结构远不止ISA和RTL(register transfer level)
  2. 关注硬件和软件的交互，设计更合适的抽象层
  3. 计算机体系结构一直被下层的技术和上层的应用塑造
    • 从历史中吸取教训应对将来的变化
  4. 软件的模式从“免费午餐”变成并行程序
    • 如何利用多核，怎么利用加速设备(GPU)
    • 新的计算模式也会对计算机体系结构产生影响
