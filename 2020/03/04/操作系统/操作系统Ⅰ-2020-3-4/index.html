操作系统Ⅰ--第一节(Chapter 1：Introduction)

发表于 2020-03-04

什么是操作系统

  1. 作为一个用户和计算机硬件之间存在的系统层的程序
    • 帮助用户执行用户的应用程序(用户程序)，在操作系统的帮助下，简化实现
    • 方便用户使用操作系统的现有功能
    • 协调计算机硬件的使用模式，管理
    • 限制了用户使用计算机的方式

操作系统结构

  1. 硬件
    • CPU、内存/外存、IO设备
  2. 操作系统
    • 控制和协调应用和用户如何使用硬件设备
  3. 应用程序
    • 应用往往本身不属于操作系统
  4. 用户
    • 人、机器、其它计算机
    • 通常使用的是应用

操作系统做什么

  1. 取决于我们怎么看
  2. 从用户的角度，通过操作系统获得操作系统使用的便易性
  3. 共享计算机如巨型机和微型机，可以同时满足多个用户的需求
    • 微型机也是远程多用户
  4. 专用系统，专用资源，基站为服务器提供服务
    • 单一用户
  5. 手持系统，某些硬件资源相对缺乏(如电池)
  6. 嵌入式系统，往往拥有很少甚至没有用户接口
    • 没有操控界面，控制其他链接的硬件按照设置的方式正常工作

操作系统的定义

  1. OS是一个资源的分配者
    • 主要指硬件资源
    • 分配上层用户同时使用资源的冲突，尽可能公平和高效
    • 提供了便捷性
  2. OS是控制程序
    • 规定了一部分，以什么方式或不可以以什么方式使用计算机
    • 是计算机在正常状态下工作，避免赖用，或不合适的使用方式
      • 防止超越权限的使用
  3. 没有通用的定义
    • 订购一个操作系统，上面提供的所有服务
    • 在计算机上一直运行的程序，即kernel
    • 时不时运行的：系统应用程序、应用程序
    • 在裸机上开发，很难实现复杂的吩咐
    • 一般至少要提供硬件驱动
      • 有些驱动程序完全工作在用户态
      • 足够的信任，商业化的驱动程序可以访问硬件

计算机启动

  1. bootstrap program(启动程序)，在通电或者重启时被加载
    • 这部分代码必须存放在启动之后，能够直接访问到的存储范围，必须在存储空间中划出一部分地址，且在上次停电后，开机之后内容不会丢掉(非易失性)
    • IO设备在计算机启动之后，是无法直接访问
    • 典型的存在ROM和EPROM(电子可编程)(不受掉电影响)，即firmware(固件，是硬件和软件的集成)
    • 通过硬件得到执行，初始化系统的方方面面
      • 使必须的硬件处于被启动的状态
    • 一次性执行，时时刻刻执行的是kernel
      • 即便只有单核的CPU，kernel同样在CPU上运行
      • kernel运行时，用户不能占用CPU
      • kernel会处于挂起状态
    • 将kernel加载，并运行kernel

计算机系统组织

  1. 从硬件上来说，CPU只能直接访问memory和cintroller直接附属的东西，无法直接访问控制器控制的硬件
    • CPU通过总线和各部分打交道
  2. 1…*个CPU和多个共享内存硬件设备，通过总线连接
  3. 通过CPU访问IO
    • 先将disk上的内容督导CPU，再写入memory
    • 通过CPU的IN OUT指令，访问disk controller
    • 读(到)内存，写IO
    • 读IO，写(到)内存
  4. CPU并行执行，设备竞争内存周期
    • IO设备和CPU本身都是以并发的方式执行功能
    • 每个设备控制器，每个都单独控制特定的设备类型
    • 设备控制器往往都有局部的缓存，存放本身工作中需要的数据
    • 为了使设备控制器能够正常工作，设备能与其他设备进行数据交换，往往通过CPU产生数据，在组成和各个不同设备控制器的局部存储间的数据移动实现
    • IO输出操作本质上是在外部设备和对应控制器的局部存储之间数据移动
      • IO只发生在控制器和相应硬件之间，controller的局部缓存
    • 设备控制器通知CPU完成了相应动作，依赖机制 —— 中断
      • CPU读取controller特定控制位的值，获取状态，得到下一步的操作

中断

  1. 由中断发生的设备产生，能够使得各种不同设备，同时通过中断同时工作
    • 迫使CPU进入中断使用状态
  2. 为了区分不同设备产生的不同类型的中断
    • 中断向量，包含服务于不同设备的中断的地址
    • 产生中断向量的方式不同：
      • 如果中断类型比较少，硬件绑定
      • 如果比较多，不可能为每一个硬件中断设计一个管脚，CPU会向外部进行查询，由外部特殊设备(中断控制器)告诉CPU发生了什么中断
      • 得到中断的入口服务地址，CPU会在入口的第一条指令开始执行
  3. 中断打断正在执行的代码，为了未来能够恢复，中断的建构必须保存正在执行的代码的当前位置
  4. trap(陷阱)和exception是由软件指令产生的中断，往往代表了执行代码中错误或者特定的用户请求
    • 是CPU内部产生的，CPU执行特定的指令所产生的中断类型
    • 软终端，区别于硬件产生的硬中断
    • CPU本身并不区分两种中断，因为响应方式是一样的
  5. 操作系统是中断驱动的
    • 一旦持续运行的kernel部分进入稳定状态，OS本身没有特别要做和特别不做的事情，只是等待中断：设备的变化和用户需求
  6. 如果只有一个单核CPU，那么运行用户程序时，很难保证会把CPU还给kernel
    • 必须强制使用户程序把CPU还给kernel
    • 这个机制就是中断
    • 定时产生中断，把CPU还给kernel，让kernel决定是否继续运行响应代码

中断的处理模式

  1. 当产生中断时，操作系统将CPU当前执行的状态保存下来，通过硬件保存
    • 主要是上下文：寄存器中当前内容和程序计数器(PC)
  2. 之后，中断服务必须确定当前中断类型，或者说是在中断向量中获取相应的地址：
    • polling(轮询)：CPU询问特定的中断硬件(设备控制器)获得中断向量，根据向量跳转到向量指向的地址入口，开始执行中断服务程序
      • 多个中断源使用同一个中断引发机制时
      • 不同的多个设备使用同一个中断
      • 比如硬盘中断，C盘、D盘、F盘…
    • vectored 中断向量：
      • 直接响应中断向量
      • 比较简单
  3. 分离的代码段决定在不同的终端下，执行相应的动作

中断时间线

  1. CPU执行不同的指令：用户程序和OS的代码
  2. IO设备：传输或空闲
  3. 通过中断产生IO设备的传输
    • 一旦传输完成，可以重新发出中断
    • 打断CPU正在执行的代码，中断服务程序处理
    • 处理完毕之后回到之前中断的位置继续执行代码
    • 随后IO可以在未来开始新的传输操作，重新发出中断
  4. CPU的入口就是每一个中断的入口
    • 中断后恢复，恢复之后处理中断前的事情，处理中断

IO结构

  1. 当一个IO的操作启动之后，直到一个IO操作完成，回到用户程序
    • wait指令，CPU空闲直到下一次中断才会返回，处理器的始终空转
    • wait loop：不断测试条件是否满足，如果测试没有满足，循环继续测试直到满足条件，经常用于检测内存访问是否满足
      • 缺点：任何时候，系统中只有一个请求得到服务
  2. IO操作启动之后哦，直接返回到后续的代码，不需要等到
    • 系统调用：用户程序发出，请求操作系统在未来时刻可以等待IO操作的具体完成
      • 系统调用是一个服务
      • 允许用户发出系统调用，操作系统做对应的事情
      • 比如read，必须保证在read完成之前不会返回到用户状态
    • 设备状态表：维护与IO相关的各种设备的信息，检测每一个设备在IO执行中的状态变化
      • 操作系统可以检查不同类型的设备的设备状态表，也根据当前硬件产生的不同状态，可以修改表象，支持用户程序对于IO调用的服务需求

直接内存访问结构(DMA)

  1. 内存是CPU可以直接访问到的存储，其他都是外存
  2. 摆脱了IO操作对CPU的依赖，直接在内存和设备间数据交互
    • 每次都通过CPU访问内存的话，最大的瓶颈在于CPU
    • IO设备和内存直接进行数据交换
    • 在CPU不控制总线时，由其他设备控制总线
  3. 由设备控制器将大批量的数据从每个设备自己的buffer中直接传输到主存，不经过CPU读出再写入的过程
    • 设备控制器可以做很多事情，但是每次的工作都是CPU告诉它做的
    • CPU可以一次告诉设备控制器做什么事情
    • 在总线空闲期间，设备控制器可以发出信号，通过总线直接访问内存
    • 这样可以一次性完成整个块的传输，而不是一个单元
    • 减少了内存访问的次数
  4. 设备只有完成整块数据传输才需要向CPU发出中断
    • 减少了中断的频率
  5. 存储并不直接连接在CPU，是通过总线连接在CPU
    • 通过总线的信号组合实现读写

存储结构

主存

  1. CPU可以直接访问到的存储
    • 一条访存指令包含地址，这个地址可以直接访问到具体的内存单元
    • 主存总是随机访问数据
  2. CPU通过指令完成各种功能，有的指令可以通过地址访问存储设备，指令中直接访问到的称为主存
  3. 随机访问：不同地址直接访问不同位置
  4. 大多数情况下，为了保证速度，都是易失性(volatile)
    • 意味着每次访问，都需要重新访问，那么变量是volatile
    • 如果一个变量被另外一个线程修改，那么相对于当前线程是volatile的
    • 如果变量可以被硬件修改，那么这个变量对于所有线程都是volatile的
  5. 其他存储都是以IO设备存在系统中

辅存

  1. 主存的扩展，可以提供更多的，容量更大的存储内容
    • 解决空间的矛盾
  2. 往往是非易失的
磁盘
  1. 多个盘面组成，运用磁技术存储
  2. 磁道
  3. 扇区
  4. 磁盘控制器：负责产生磁盘的相应操作
    • buffer写入磁盘
    • 磁盘读到buffer
  5. 问题在于：必须通过访问特定扇区的号，访问特定的数据
    • 磁盘的访问必须有相应的机械硬件访问
    • 速度较慢
    • 这些动作通过controller完成
固态硬盘(SSD)
  1. 比磁盘更快，非易失性的
    • 时延短
    • 类似内存的访问速度
    • 不同的工艺实现
  2. 现在已经有无磁盘的系统

存储架构

  1. 存储系统被分层组织：
    • 速度
    • 成本
    • 易失性
  2. 为什么主存做成volatile
    • nonvolatiele使得写入不会轻松丢失，需要更大的电流，更多的时间
    • 大量在主存中使用非常小电流的写操作
    • 辅存本身就是为了存储数据，在系统的启动之间保存数据，不会丢失
  3. caching
    • 可以将一些数据信息从一个层次的存储中拷贝到速度更快的存储中
    • 作用：CPU不能直接访问到所有辅存，可以辅存中部分内容存到主存中，辅存就是主存的caching
  4. 设备驱动
    • 为不同类型的设备控制器提供特定的代码进行IO管理
    • 实现了一个框架，使得各种各样不同类型的设备控制器和操作系统内核以一种比较统一的方式进行交互
      • 即可以抹平不同设备控制请间的差异，简化内核和设备控制器交互的细节

存储的层次结构

  • 寄存器：本身构成了CPU的一部分
  • cache：是透明的，在CPU看来不存在，提供主存更好的性能
  • 主存：支持CPU随机读写
  • SSD
  • 磁盘
  • 光盘
  • 磁带
caching
  1. 硬件里面和操作系统以及软件都可以利用caching进行提速
  2. 主要作用把位于比较慢速的存储中的信息移动(复制)到比较快速的存储中
    • 先在快速设备中检查数据是否存在
    • 不存在的话访问慢速设备
  3. 有时候不能直接访问慢速设备，如CPU没有能力直接访问磁盘、SSD
  4. cache小于被cache的存储大小
    • cache的组织大于设计问题
    • cache大小和替换策略
