计算机体系结构--第三节(处理器：datapath and control)

发表于 2020-03-19

Recap：Computer history

  1. 计算机体系结构 >>(远远不止) ISAs 和 RTL(电路)
  2. 体系结构的形成由计算机基于的技术和我们面临的应用一起打造的
    • 计算机的历史对我们有帮助
  3. 从Babbage到IBM360，有130年的历史
    • 计算机：早期类似计算器(没有分支跳转) –> 现在完全可编程
    • 第二次世界大战推动了计算机发展：电子机械方式 –> 纯电子
  4. 软件开发的代价，受限于大量体系结构的影响
    • 解决兼容性问题
  5. IBM360引入”机器族”的概念
    • 相同的指令集
    • 生产不同，实现方式不同
    • 1964年，一天同时发布了6个不同的型号
    • 这些架构设计在后续模型中依然在使用

Recap：ISA

  1. 提供了软件和硬件间的接口
  2. 一般来说，指令集提供了
    • 程序员可见的状态(寄存器+内存)
    • 指令，操作这些状态
  3. IBM360是第一种将指令集和底层实现分开的机器
    • 即 microarchitecture
  4. ISA有很多可能的实现，比如：
    • 苏联人制造了IBM360的代码兼容克隆，Amdahl离开IBM后也是如此
    • 如今，你可以买到运行x86-64 ISA的AMD或英特尔(Intel)处理器。
    • 很多手机使用ARM ISA，苹果、高通、三星、华为等多家公司都有ARM ISA的实现。
  5. ISA通常与下层的微体系结构风格有关，比如
    • 累加器：硬件、没有流水线的、一块完整的电路
    • CISC：microcoded(微编码的)
    • 【问】微编码是什么意思？
    • RISC：固化的、流水线的方式
    • VLIW：时延固定，指令顺序固定，并行的流水线
    • JVM：软件解释的方式实现
  6. 但是这些体系结构都可以用任意微体系结构方式实现
    • Intel Ivy Bridge：固化的流水线CISC机器
    • Spike：RISC-V指令集的软件解释器
    • ARM Jazelle：JVM的硬件执行方式，专门用来执行Java的字节码

Recap：ISA – 必须明确的有哪些？

  1. 指令执行的过程
    • 指令获取
    • 指令译码
    • 操作数的读取
    • 指令的执行
    • 结果的写回
    • 获取下一条指令
  2. 需要考虑的问题：
    • 指令格式或解码
      • 是怎么解码的？
    • 操作数和结果的位置
    • 数据类型和长度
    • 运算
      • 支持哪些运算
    • 如何获取后续的指令
    • 指令选择，实现那些指令
    • 寻址模式
      • 数据寻址模式
      • 偏移的大小
      • 立即数长度
    • 操作数宽度
    • 分支指令
    • 指令格式

ReCap:MIPS 的显著特征

  1. MIPS比RISC-V早很多，是早期的RISC架构
  2. 支持multi-circle
  3. 32-bit固定格式指令
  4. 32个32-bit 通用寄存器(R0 = 0；R31 放返回地址)
  5. 32个浮点数寄存器
  6. 三地址的寄存器-寄存器算数指令
  7. load/store都是单地址的方式：基地址+偏移
    • 没有 indirection 和 scaled 方式
  8. 16-bit立即数
  9. 简单的分支条件
    • 与零比较或者两个寄存器比：=，≠
    • 没有condition codes
  10. 延时选择分支
    • 无论分支是否发生，都是在分支选择或者发生跳转之后执行指令
    • 编译器可以找到必然执行的指令放在延时分支后面，有50%的几率可以找到必然执行的指令

The Big Picture:The Performance Perspective

  1. Performance of a machine is determined by:
    • Instruction count
    • Clock cycle time
    • Clock eycles per instruction
    • 【问】前两项有什么区别？
  2. Processor design(datapath and control)will determine:
    • Clock cycle time
    • Clock cycles per instruction

如何设计处理器

  1. 分析指令集 –> datapath 需求
  2. 选择 datapath 组件，建立时钟控制
  3. 组合 datapath 满足 1 中的需求
  4. 分析每条指令的需求，确定影响寄存器传输的控制点
  5. 组合控制逻辑

以MIPS为例(32-bit)

  1. 所有MIPS指令都是32-bit：

    • shamt：shift amount

step 1a：分析指令集

  1. 所有指令执行都是开始于取指

step 1：指令集的需求

  1. 内存
    • 指令 & 数据
  2. 寄存器(32 × 32)
    • 读 rs 寄存器
    • 读 rt 寄存器
    • 写 rt 或 rd 寄存器
  3. PC
  4. 有符号/无符号扩展
  5. ALU：加、减、扩展立即数
  6. PC+4 或者 PC+ext[imm]

step 2：datapath的组件

  1. 组合电路的基本单元
  2. 存储单元
    • 内存
    • 寄存器
  3. CLock Skew
  4. 对组合电路时间的要求：
    • (CLK-to-Q + Shortest Delay Path) > Hold Time
    • 否则上一个输出值没有被记录下来
    • Cycle Time = CLK-to-Q + Longest Delay Path + Setup + Clock Skew
    • CLK-to-Q + Shortest Delay Path > Hold Time + Clock Skew

step 3：Assemble(组装) DataPath meeting our requirements

  1. RTL 的需求 –> 组装datapath
    • 指令的获取
    • 操作数的读取和操作的执行
