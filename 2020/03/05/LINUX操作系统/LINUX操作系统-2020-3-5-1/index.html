LINUX操作系统--第二节(图形界面，不考)

发表于 2020-03-05

图形用户界面GUI

X Window: C/S模型

  1. LINUX的桌面环境是基于X Window协议的
  2. 窗口系统并不在内核中实现，可以运行在不同的操作系统以及不同类型的计算机、显卡、显示器和鼠标设备等之上
    • 基于传统的X窗口协议，一种可移植性的、网络化、且与硬件无关的图形窗口系统
    • Wayland协议试图替代X窗口协议
    • 不在内核中实现，可以随时安装不同的窗口系统
  3. 采用C/S模型，从X的应用角度，X服务器为X客户(应用)提供显示和输入服务
    • X应用例如：firefox，xterm
    • X Server利用内核接口从输入设备接收输入并发送给X Client –> 根据X Client的图形输出请求，在输出设备(显示器)上绘制
    • X Client和X Server可以在同一台主机上或者不同主机上(通过TCP连接，端口号6000)
    • 一个X Server上可以同时有多个X Client，这些Client可来自于不同的主机
    • X Client的基本操作单元称为窗口(Window)，一个窗口对应着屏幕上的某个有边界的区域
    • 类比：X Server提供了多块虚拟的画板，也就是window，X Client可以在这块虚拟的画板上绘制，X Server负责将画板呈现在自己的显示器上
  4. 传统网络应用 VS X Window
    • 传统网络应用：服务器在远端，客户在本地
    • X Window：X服务器在本地，客户在远端

X Window：窗口管理器和显示管理器

窗口管理器：

  1. 一个特殊的X Client，负责所有X Client对应的窗口的管理，控制着窗口和窗口相关图形元素的外观和特征
    • 将内容绘制到画板/窗口上
    • X Server将X Client绘制的窗口显示出来(通过窗口管理器)
  2. 改变各个窗口的大小和位置，处理窗口重叠的情况，控制窗口属性(滚动条、图标和菜单等)
  3. 支持用户可定义的快捷键和鼠标操作
  4. 提供任务栏、桌面背景及虚拟桌面功能
  5. Ubuntu最新的窗口管理器缺省为gnome-shell(GNOME3)

显示管理器：

  1. 替代文字终端的getty，提供图形化的登陆界面
    • 缺省只能管理本地的X服务器以及WayLand服务器
    • 也可通过XDMCP协议管理多个远端的X服务器
    • 用户可以选择登陆后启动哪一个桌面环境
  2. Ubuntu缺省的显示管理器为：GNOME3提供的gdm3

环境变量DISPLAY

  1. X应用要连接到哪个X Server？一般支持display选项(–display )，或设置环境变量DISPLAY

  2. Ubuntu下的桌面环境或X服务器缺省不允许远程X应用通过TCP协议访问，只允许本地X应用访问

  3. 每个X服务器可通过hostname:displaynumber.screennumber来描述

    • hostname：X服务器所在的主机名或者IP地址，如果省略，则表示本机

    • displaynumber：显示器号码，所谓display指的是共享键盘、鼠标等输入设备的一组monitor(虚拟的显示器)

      • 大多数主机一般仅开启一个X服务器，但也可以在多个虚拟控制台开启X服务器，通过显示器号区分
      • 缺省为0，后面的X服务器的显示器号码一般逐步递增，也可人工指定

    • screennumber号是可选的，缺省为0时不包括。如果一个主机连接有多个屏幕时，则可通过屏幕号来决定使用哪个屏幕

补充：访问控制 —— xhost

  1. 大部分情况下，X Server和X Client都是在同一个主机上面，不需要特别的设置，也不需要X host命令
  2. 远程的访问，一般会选择通过ssh协议建立隧道进行中转

桌面环境

  1. 把日常工作中经常使用的那些应用程序紧密地集成在一起
    • 提供一个一致的拥有标准外观和体验的用户界面
    • 提供图标、任务栏、墙纸、桌面小控件等
    • 包括寻找和启动一个应用程序，系统配置，查找和编辑文件，办公文档处理，网络浏览和邮件阅读等功能
    • 支持应用程序之间的通信，包括支持拖放操作等
  2. 1996年第一个基于X的桌面环境的项目KED(最初命名为Kool Desktop Environmental，后来表示K Desktop Environment)由一名德国大学生Matthias Ettrich启动，基于Qt库开发
  3. 由于早期Qt库的许可证方面的原因，1997年GNOME(GNU Network Object Model Environment)项目启动，基于GTK+库创建另一个桌面环境。目前有GNOME2和GNOME3
  4. Xfce是一个更加轻量级的桌面环境，也是基于GTK+。其他常用的桌面环境还包括轻量级的LXDE、Mint的MATE等

GNOME3桌面环境

  1. 顶部面板条
    • 活动：窗口切换器，搜索应用，切换工作区，当前工作区活跃的应用
    • 应用程序菜单：当前活跃的应用所提供的菜单项
    • 日历和通知
    • 系统菜单：系统设置，切换用户和注销退出、关机和重启等
  2. Dock浮动面板，可调整在左右和底部
    • 应用按钮
    • 常用应用列表(收藏夹)
      • 图标左边小矩形表示该应用活跃
      • 可固定某些应用：拖拽或右键添加到收藏集

GNOME3：活动预览

  1. Super键(Windows键)和Esc键

GNOME3：启动应用程序

  1. 快捷键ALT+F2，运行命令：要求知道应用名字的全名
  2. 终端中运行命令，可以利用bash的自动补全功能

GNOME Shell扩展

  1. 建议安装GNOME Tweak Tool(优化)，可以调整GNOME本身的一些高级设置，可以管理主题、扩展、字体、键盘和鼠标、电源、自启动应用等

  2. GNOME shell扩展提供对桌面环境的扩展，扩展可通过顶部面板条访问

  3. 访问 https://extensions.gnome.org 以选择扩展安装，安装的扩展一般保存在目录~/.local/share/gnome-shell/extensions中

GNOME Shell：常用扩展
  1. 扩展名 URL
    Dash to Dock https://extensions.gnome.org/extension/307/dash-to-dock/
    User Themes https://extensions.gnome.org/extension/19/user-themes/ 主题可以在gnome-look.org下载
    Gno-Menu https://extensions.gnome.org/extension/608/gnomenu/ 经典样式Gnome应用程序菜单
    Removable Drive Menu https://extensions.gnome.org/extension/7/removable-drive-menu/ 可移除磁盘控制
    Clipboard Indicator https://extensions.gnome.org/extension/779/clipboard-indicator/ 剪贴板管理
  2. 关闭扩展
    • 通过gnome-tweaks将扩展暂时关闭
    • 在~/.local/share/gnome-shell/extensions中找到相应的扩展，将其删除
    • 通过gnome-software，选择“已安装”，移除指定的扩展
