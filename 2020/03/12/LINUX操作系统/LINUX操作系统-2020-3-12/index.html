LINUX操作系统--第三节(文件系统初步)

发表于 2020-03-12

终端组合键

  1. CTRL-\ 退出进程，将对应内存状态保存到core文件中
  2. CTRL-C 和 CTRL-\ 都有可能被忽略，但是 CTRL-Z 无法忽略，fg 命令将最近的后台命令转为前台
  3. CTRL-S 表示暂停屏幕输出，CTRL-Q 表示恢复屏幕输出
    • 建议关闭 CTRL-S/Q：stty -ixon -ixoff
    • 建议将其添加到bash配置文件：用户HOME目录下的.bashrc中
  4. 输入特殊字符：首先按<CTRL-V>然后按特殊按键，比如<CTRL-C>

bash组合键：man readline

  1. bash使用readline库支持命令行输入和编辑，可以使用readline快捷键

  2. 内置命令bind可查看和设置readline键映射及readline变量

    • bind -p列出快捷键和相应的功能
    • \C表示ctrl，\e表示ESC

  3. /etc/inputrc和~/.inputrc: readline的设置和键映射的配置文件

    • 可以用来设置readline映射
    • ~/.inputrc是readline对应的配置文件，可以用来设计readline库和对应键的映射
      • 新建~/.inputrc，之后参照/etc/inputrc添加映射

  4. bash组合键

    • 带*为终端组合键，在其他地方也可以使用

文件系统基础

  1. 文件是用于存储信息的容器
  2. Linux系统中，文件是一个广泛的定义
    • 可以是保存在硬盘、U盘或者光盘上的数据
    • 可以是键盘、鼠标、显示器、打印机等物理设备
    • 可以是系统虚拟的文件等，如内核维护的相关信息
  3. 每个文件有一个名字，可通过标准的(读、写和ioctl)接口访问文件
  4. 文件系统：将文件按照一定的方式组织和管理
    • 引入“目录”的概念，将整个Linux文件系统组织成一颗大的层次结构的目录树
    • 目录(directory)也是特殊类型的文件，用来“包含”文件：
      • 有时也称为文件夹(File Folder)
      • 目录还可以包含其他目录
      • 目录维护了如何组织和访问其他文件以及子目录的信息

文件系统

  1. Windows操作系统中可以有多个目录树，不同的磁盘分区对应不同的目录树
  2. Linux操作系统以唯一目录树的形式来组织
  3. 根节点称为根(root)目录，用/来表示
  4. 目录树中的名字只需在其所在目录中唯一
  5. 路径名(pathname): 路径分隔符为/
    • 绝对路径名：以/开始，从根目录开始逐级子目录寻找直到到达最终的目录项。如 /home/apple/linux/quizs/quiz1.pdf
    • 相对路径名：不以/开始，从当前工作目录开始
      • 为用户维护一个工作目录，登录时被自动设置为该用户的主目录(在用户被设置时创建，/etc/passwd 文件)
      • 假设当前工作目录为 /home/apple ，相对路径名可以为 linux/quizs/quiz1.pdf
    • 允许出现多个连续的 / ，相当于一个/，比如 //home////apple 等价于 /home/apple
  6. 每个目录文件包括两个特殊的目录项
    • .(单个点)表示当前目录
    • ..(双点)表示当前目录的父目录，如果当前目录已经是根目录，则其父目录也是根目录本身
    • 不管是绝对路径还是相对路径，当前目录(.)和父目录(..)都可在路径名中出现 /home/apple/linux/../../apple/linux/./quizs/quiz1.pdf 相当于 /home/apple/linux/quizs/quiz1.pdf
    • 假设当前工作目录为 /home/peach ，../apple/linux/quizs/quiz1.pdf 相当于 /home/apple/linux/quizs/quiz1.pdf

路径名解析相关指令：dirname/basename/realpath

  1. dirname pathname...：
    • 返回路径名(pathname)中除了最后一个名字外前面的部分，即所在目录
  2. basename pathname...：
    • 返回路径名(pathname)中最后一个名字，即文件名
  3. realpath [OPTIONS] pathname...：
    • 返回路径名解析后的绝对路径名，缺省除了最后一个部分外前面的名字都必须存在
    • -e, --canonicalize-existing 路径名中的名字必须实际存在
    • -m, --canonicalize-missing 不检查路径名中的名字是否存在

改变目录和波浪号扩展

  1. pwd命令: 查看当前工作目录，print working directory
  2. cd [dir]: 改变工作目录到dir
    • cd后面没有参数时表示工作目录设置为用户主目录
    • 参数为-，表示工作目录设置为先前的工作目录
  3. 如果希望在多个目录间切换
    • 内置命令pushd/popd维护一个目录栈
    • dirs命令查看目录栈
    • help [topic] 查看bash内置命令的帮助信息
  4. 也可以对目录设置别名
  5. shell支持波浪号扩展(Tilde expansion)
    • 单独的波浪号~表示当前用户的主目录，而~user表示用户user的主目录
      • 如果用户名不存在时，波浪号扩展无法展开，仍然是~user
    • 波浪号扩展应该出现在一个单词(word)开始，且接下来是单词结束或者/
  6. set -x 打开bash的xtrace功能, 在执行命令前将命令输出
    • set +x 关闭bash的xtrace功能

改变文件时间命令touch

  1. 创建新的空文件：如果文件FILE不存在, 缺省情况下touch会创建一个长度为0的普通文件
  2. 改变文件的时间：如果FILE存在, 内容不变, 仅改变时间。缺省修改和访问时间都设置为当前时刻
  3. 每个文件(目录等)保存的时间信息包括：
    • 修改时间mtime: 修改文件内容时更新
    • 访问时间atime: 访问(读)文件时更新
    • 状态改变时间ctime: 文件内容或元信息(如权限/名字/时间等)改变时更新, 用户无法改变ctime，而是内核自动更新
    • 【疑问】ctime不可以由用户改变吗
  4. Mac OS还保存了出生时间birth time，即创建该文件的时间
  5. 修改文件时一般mtime/atime/ctime都更新，读文件一般仅atime更新
  6. 用途：
    • 修改时间，强制make重新编译文件
    • 备份某一个时间之后修改过的文件，先设置一个BASE时间

文件名格式

  1. Linux文件名(包括目录等)区分大小写，最长255个字符，路径名最长4096个字符
  2. 文件名不能命名为单点和双点，也不能包含斜杠/和NULL字符（ASCII码为0）字符，除此之外可以包含任何字符
  3. 虽然可以使用后缀，但与windows系统中严重依赖文件后缀不同，Linux对此并没有任何的假设
  4. 不建议包含特殊字符，而是建议仅使用字母数字以及点(.)、连字符(-)和下划线(_)，且避免第一个字符为连字符，也不建议包含空格
    • 如果第一个字符为连字符，在作为参数时可能会被命令解释为短选项
    • 【疑问】：touch -apple 为啥不会被解析为 touch -a pple
    • 如果文件名中包含了空格(如1 quiz1.txt)，在作为参数时可能会被解释为两个以上的参数(1和quiz1.txt)
  5. 如果要包含shell扩展中用到的字符(如空格*&|><等)，文件名参数可以用单(双)引号括起来，表示不进行扩展

列目录命令ls

  1. Linux系统中使用最多的命令

  2. 显示FILE对应的文件或目录的信息

    • 后面不带参数，表示列出当前目录(.)的内容
    • 如果FILE是目录时会缺省会列出该目录包含的内容

  3. 缺省情况下:

    • 以短格式输出，仅仅显示文件或者目录名，而不会显示详细信息
    • 不会显示那些以.开始的文件名（隐藏文件）
    • 显示的内容按照locale环境设置根据文件名进行排序，一般为字典顺序
      • 字典顺序：按照0-9…aAbBcC…zZ等顺序，这样不管大小写，那些相同前缀的名字排列在一起
    • 如果标准输出为终端，则缺省多列显示(-C)
    • 如果标准输出不是终端，则缺省每行一个文件(-1)

  4. 隐藏文件：那些名字以点(.)开始的的文件或目录, ls缺省不会显示，缓存目录或者临时目录

    • -a 选项：显示隐藏文件，包括当前目录(.)和父目录(..)
    • -A 选项：与-a类似，但是当前目录和父目录不会显示

  5. -F, --classify: 文件名后面添加类型指示 (*/=>@|)

    • 区分文件类型
    • -l 含义 -F附加
      - 普通文件 *可执行
      d 目录 /
      b 块设备文件
      c 字符设备文件
      l 符号连接 @
      p FIFO |
      s socket =

  6. -l 长列表方式显示

    • 参数为目录时缺省列出该目录的内容
    • drwxr-xr-x | 2 | root | root | 4096 | Feb 23 14:19 | bin
      |–|–|–|–|–|–|–|
      文件类型+访问模式 | 引用计数 | 拥有者 | 用户组 | 文件大小，字节 (-h友好方式显示) | 修改时间 | 文件名

  7. -d, --directory 显示目录本身的信息

    • -d选项经常与-l选项一起使用

  8. -R, --recursive (深度优先)递归列出目录及子目录内容

  9. 长列表时缺省显示修改时间：

    • 如果时间为最近6个月，则不包括年份但是包括小时分钟
    • 如果超过6个月则显示包括年份在内的日期
    • ls –l 缺省显示的是修改时间
    • ls -lu 显示访问时间(use time)
    • ls –lc 显示状态改变时间(change time)

  10. -i, --inode 显示inode编号

  11. 排序控制: 缺省文件名先后顺序排序

    • -r, --reverse 反向排序
    • -X, --sort=extension 按照后缀排序，没有后缀的优先
    • -S, --sort=size 按照文件大小排序，缺省从大到小
    • -t, --sort=time 按照时间排序，缺省为修改时间，最新优先(从新到旧）
    • -ltc 状态改变时间排序，显示的为状态改变时间
    • -ltu, 访问时间排序，显示的为访问时间
    • -U, --sort=none 不排序 （目录顺序）
    • ls –l 按照文件名排序
    • ls –lX 按照文件后缀顺序排列
    • ls –lSr 按照文件大小从小到大的顺序排列
    • ls –ltr 按照修改时间从旧到新的顺序排序
    • ls –ltu 按照访问时间从新到旧的顺序排序
    • ls –ltcr 按照状态改变时间从旧到新的顺序排序

  13. 按照大小顺序从大到小排列， head取前面6行， sed 1d去掉最前面的一行，显示前面5个最大的文件：ls -lS / | head -6 | sed 1d

tree 查看目录树

  1. Ubuntu缺省没有包含tree命令，可通过sudo apt install tree安装
  2. 以树的方式列出目录树中某个分支的内容，缺省通过缩进方式显示，彩色输出(使用环境变量LS_COLORS的设置)，也可输出为HTML/XML或JSON格式
  3. 选项 作用
    -d 仅仅列出目录，缺省包含目录和文件
    -L level 显示几层目录
    -x 表示不允许跨越到其他文件系统
    -a 允许隐藏目录和文件，缺省不包含在内
    -H base 输出为HTML格式，目录和文件的超链接最前面为base
    -X 输出XML格式
    -J 输出JSON格式
    -l 如果符号连接指向目录，则展开符号连接
    -I pattern 不列出与模式pattern(可使用标准通配符加上表示任一模式的
    -F 文件名后添加类型指示

创建目录mkdir

  1. 同一个目录中不允许有两个相同名字的目录项，
  2. 缺省情况下如果父目录不存在时，不能创建子目录
    • 从目录树的角度而言，不允许创建一个没有长在树上的新树枝。
    • -p选项（make parent）如路径上的目录如果不存在，会自动创建途中所有需要的父目录

删除目录 rmdir

  1. 仅允许删除空目录
  2. 如果想一次性移除一串空目录，可采用-p选项（delete parent），会自动移除途中所有可以移除（为空）的父目录，rmdir -p dir1/dir2/dir3 相当于调用 ( rmdir dir1/dir2/dir3;rmdir dir1/dir2; rmdir dir1 )
  3. 如果要递归删除目录下所有内容怎么办？
    • rm –R courses 会删除courses开始的子树

复制文件和目录命令cp

  1. 复制文件：复制某个文件并命名为新的名字
  2. 复制文件：如果目的目录存在，复制一个或多个文件到另一个目录，名字不变
  3. 复制目录：要使用-r/-R选项, 将目录递归复制到另一个目录中(可以改名)
  4. 是1还是2？ 如果最后一个参数存在且为目录Directory时，表示格式2，即将前面的一个或多个文件复制到该目录中
  5. 如果要复制的位置存在相同的文件名，权限允许的情况下缺省会直接覆盖(写得权限)
  6. 复制目录时，要求采用-r(-R)选项，表示递归复制目录下的文件以及子目录，即将目录下的全部内容复制

复制文件和目录命令cp: 覆盖和备份

  1. 如果要复制的位置存在相同的文件名，权限允许(即可以写方式打开）时缺省会直接覆盖。权限不允许时会报Permission denied
    • -f, --force 表示在文件存在时不询问用户而强制覆盖(先remove然后复制)
    • -i, --interactive 在覆盖已有文件时询问用户
    • -n, --no-clobber 表示不覆盖已有文件
    • -b 已有文件存在而覆盖时保留副本，缺省后缀为~，可通过-S, --suffix=SUFFIX 指定其他后缀
      • 缺省只会保留一个副本，第二次会覆盖
    • --u, --update 常用在进行备份时，只有在目的文件不存在（不需要覆盖）或要被覆盖的文件存在但是修改时间更老时才会复制文件
    • --backup[=CONTROL]: 文件存在时首先备份，给出了如何备份，具体细节参考文档
  2. 复制文件相当于创建一个新的文件，缺省情况下该文件的拥有者、权限、时间等为创建文件的缺省值，即为当前用户及当前时刻
    • -p选项（preserve）:如果有可能，保留原文件的使用者的身份及权限以及时间属性等

cp:复制目录结构和内容

  1. 复制目录时，要求采用 -R 选项或 -a 选项
    • -R, -r, --recursive 递归复制目录下的文件及子目录到指定目录
    • -a, --archive 归档,递归复制，保存原文件的各个属性
  2. 目的目录是否存在时，复制目录的行为不一样！
    • 如果目的地存在且为目录，则表示将前面的文件和目录递归复制到该目录中
    • 如果目的地不存在，则表示复制目录并改名
      • 相当于首先 mkdir 一个目录，然后使用 -R 选项将源目录中的所有内容再递归复制到这个新建的目录中
  3. 在复制目录时，如果其中的某些目录已经有内容：
    • Linux下的目录复制为合并(merge)操作，只有相同的文件会被丢掉
    • Mac OS为replace操作，需要特别注意

移动或重命名命令mv

  1. 如果新的位置已经有相同的名字(文件或目录)存在，则覆盖原有的文件或目录
  2. 只能将文件覆盖成文件，或将目录覆盖成目录，且在覆盖目录时要求被覆盖的目录为空目录
  3. 缺省情况下覆盖时不询问用户，所以要谨慎使用
  4. 选项 动作
    -t, --target-directory=DIRECTORY 移动到指定目录
    -i, --interactive 在文件存在时询问是否覆盖
    -f, --force 在文件存在不询问用户直接覆盖
    -n, --no-clobber 不覆盖已有文件
    -b 若需覆盖文件，则覆盖前先行备份，备份文件后缀缺省为~
    -S, --suffix=SUFFIX -S, --suffix=SUFFIX
    -u, --update 目标文件不存在，或目标文件存在但是比源文件旧时才会移动
    -v, --verbose 解释要执行的操作

删除命令rm

  1. 缺省删除文件, 通过-R选项也可以删除目录
  2. 缺省会不询问就删除文件，但在文件不可写时且为交互式shell时会询问用户是否删除
    • 和mv类似
  3. 一旦删除成功，则需要用到文件系统修复工具才有可能找回删除的文件
    • ext2/ext3/ext4类型的文件系统可以采用第三方 extundelete
    • ext2(ext3/ext4?)类型文件系统也可用系统自带的交互式文件系统调试工具debugfs
  4. shred命令可以删除文件（用一些随机数据来覆盖原文件的内容），且很难恢复
  5. 选项
    -i 删除前每次询问用户
    -f, --force 不询问用户就删除
    -R, -r, --recursive 删除目录及子目录下的所有内容
    -d, --dir 删除空目录
    -v, --verbose 解释要执行的操作

我的答案

  1. 删除字符：Ctrl-H 或
    删除单词：Ctrl-W
    删除一行：Ctrl-U
  2. 不能被忽略的快捷键：Ctrl-Z
    作用：将进程转到后台运行
  3. 文件名不能命名为单点和双点，也不能包含斜杠/和NULL字符
  4. a. cd ~
    b. pwd
  5. a. cd ../../mail
    b. cd ../..
  6. ls -d 显示的父目录本身的信息，而 ls 显示的是父目录的内容
  7. a. -a
    b. -r
    c. -R
    d. -F
    d. -lh
  8. ls -lu按照文件名的字典序排序，ls -lut按照访问时间从新到旧的顺序排序
  9. a. mkdir -p books/literature/english/shakespears
    b. touch books/literature/english/shakespears/hamlet.txt
    c. rm -rf books/literature
  10. cp -r bar1 bar2
    bar2不存在：先创建目录bar2，之后递归复制bar1中的所有文件和目录到新建的bar2目录下
    bar2存在：会将bar1中的所有文件和目录，连同bar1目录文件一起复制到bar2下
  11. mv bar1 bar2
    bar2不存在：相当于对bar1重命名为bar2
    bar2存在：移动bar1到bar2，如果bar2下本身存在bar1，会覆盖
