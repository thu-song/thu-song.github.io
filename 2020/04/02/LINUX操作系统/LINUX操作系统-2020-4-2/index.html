我的答案

  1. which cmd
    type cmd
  2. help case
  3. /etc/bask.bashrc和.bashrc
  4. source ~/.bashrc
  5. 将当前的时间信息附加到tmp目录下的一个文件中，文件名为tmp，后缀名为当前的日期，月-日的格式
  • 非交互式
  • 非登录shell
  • 会执行~/.bashrc
  1. mkdir -p {c,python}/{slides,homeworks,quizs}
  2. touch tmp{0..09}.txt
  3. #! /bin/bash
    read -p “请输入你的名字: “ name;
    echo “欢迎$name来到LINUX世界。”
  4. echo $(date; ls /)，打印当前的时间和/目录下的文件名，没有装引号会进行单词分割，IFS缺省情况下所有单词之间都以一个空格隔开
    echo “$(date; ls /)”，打印当前的时间和/目录下的文件名，会保留原有的(date; ls /)输出的格式打印在屏幕上，不会进行单词分割处理
  • 如果foo1执行失败，那么之后的命令都不会被执行；最终状态码为1
  • 如果foo1执行成功，foo2会被执行，并且foo2的输出内容将通过管道输入到foo3，之后将foo3的输出会被重定向到foo4文件中；最终状态码取决于foo3，如果foo3执行成功，状态码为1，否则为0

参考答案

  1. 要找出外部命令cmd所在的位置，请给出两种方法。
    type cmd or type -P cmd
    which cmd
    whereis cmd也可以

  2. 我们知道bash支持选择结构和循环结构，如果想要查看case语句的用法，应该使用什么命令？
    help case or help -s case

  3. 通过图形界面启动终端应用，会执行哪些shell的配置文件？
    不是登录shell，是交互式shell，会执行环境文件，即系统和用户环境文件 /etc/bash.bashrc 和~/.bashrc，
    注意先执行系统配置文件，然后用户配置文件

  4. 修改了用户配置文件.bashrc之后，应该执行什么命令使得用户的修改马上起作用？
    source ~/.bashrc 或者 . ~/.bashrc

  5. 执行如下命令，请给出该命令做了什么事情。bash执行时是交互方式还是非交互式方式？是登录shell方式还是非登录shell方式，会执行用户主目录下的.bashrc吗？
    bash -c ‘date >> /tmp/tmp.$(date +%m-%d)’
    执行date命令，其输出附加到/tmp目录下的文件tmp.xxxx，xxxx为当天的信息，即month-day
    -c 表示采用非交互方式，除非有-i选项。 非登录shell。采用非交互方式执行，这样不会执行主目录下的.bashrc。
    如同执行脚本一样，也是非交互式方式，显然不应该执行.bashrc。

  6. 假设要在当前目录下创建如下目录结构：
    c/slides/
    c/homeworks/
    c/quizs/
    python/slides/
    python/homeworks/
    python/quizs/
    请用一条语句，尽量少的文本来完成上述任务。

    mkdir -p {c,python}/{slides,homeworks,quizs}, 注意一定要选项-p

  7. 要在当前目录下创建10个空的普通文件，分别命名为tmp00.txt,tmp01.txt,….,tmp09.txt，请用一条语句，尽量少的文本来完成上述任务。
    touch tmp{0..09}.txt

  8. 请写一个简单的脚本，询问用户的名字，然后输出：欢迎某某来到Linux世界。

    #! /usr/bin/env
    read -p ‘请输入您的名字…’ name
    echo “欢迎${name}来到Linux世界”

#! /usr/bin/env
read -p ‘Please input your name…’ name
echo “Hello ${name}, welcome to the Linux world.”

注意 bash read -p …. 没有外部命令read，而且采用bash read…是启动一个新的子进程，即便得到输入，但是也无法反馈回父shell

  1. 请简单描述下面两条命令执行时有什么区别？
    echo $(date; ls /)
    echo “$(date; ls /)”

    命令替换后要进行单词分隔，多个参数传递给echo，参数之间以空格隔开，所以看到的是一行的内容
    双引号允许命令替换，但不会单词分隔，作为一个整体传递给echo，所以看到的是多行内容

  2. 假设pipefail选项关闭， 请简单描述一下这个命令的执行步骤
    ! foo1 && foo2 | foo3 > foo4

foo1执行，如果成功，由于!改变状态码 ，即! foo1 失败，短路逻辑，后面的命令不执行，最后的状态码为1
foo1执行，如果失败，! 改变状态码，! foo1 成功，执行后面的命令，即 foo2 | foo3 > foo4，
也就是执行foo2，其输出由foo3处理，最后重定向到foo4，最后的状态码为foo3命令返回的状态码

在程序语言中，运算符的优先级是这样的，先算再比，比完后进行逻辑运算，在逻辑运算中一元逻辑运算优先级更高
也就是说 算术运算符优先级最高，比较运算符次之，然后是逻辑运算符
shell也是类似的策略， 这里管道可以看成算术运算符，这样运算顺序是这样的，我这里用括号来给出运算顺序
( ! foo1) && (foo2|foo3 >foo4)
