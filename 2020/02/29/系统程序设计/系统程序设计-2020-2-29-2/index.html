系统程序设计--第二节(File IO)

发表于 2020-02-29
  1. file IO function:
    • read, write, open. close. ioctl, …
  2. C/C++/Java等中也有IO相关的接口
    • 这些语言封装了底层的IO操作
  3. 尽量使用高级语言中的流/带有缓冲的IO接口
    • 当我们做底层开发时，使用底层的IO接口
    • 写kernel接口时，需要了解底层的IO接口的规律

File Descriptor

  1. 所有打开的文件都用文件买哦舒服表示
  2. 0，1，2分别表示标准输入、标准输出和标准错误

data structure

  1. user file descriptor table –> file table –> inode table
  2. 读写位置存在file table中
  3. 每个进程都有一个自己的fd表，整个系统只有一个file table和inode table
  4. fd表的结构是固定的，限制了一个进程可以同时打开的文件数目
  5. 操作系统并不会为文件操作提供同步操作

open

  1. man 2 open
  2. return：成功返回文件描述符，失败返回-1
  3. 选项O_SYNC，写操作会先写到缓冲区，再写到磁盘
  4. file table中每次利用open打开都是不同的，具有独立的游标
  5. 比较老的版本的操作系统支持的文件名长度小，会直接把多余部分截掉
    • 可以通过指定宏 POSIX_NO_TRUNC 避免

creat

  1. int creat(const char *pathname, mode_t mode);
    • Return: 成功创建一个只读的文件返回文件描述符，失败返回-1
  2. 等价于指定O_CREAT选项的open操作
    • open(pathname, O_REONLY | O_CREAT | O_TRUNC, mode);

close

  1. close(fd)
  2. 释放利用的数据结构和指针，以及文件上的锁(?:加了锁的情况下可以被close吗？)
  3. 检查被打开文件的link count，如果该值为0，该文件会被回收
    • 打开文件后，把文件删除，文件不会立即消失，关闭之后才会被彻底回收
  4. 当进程结束时，所有打开的文件都会自动被kernel回收

lseek

  1. off_t lseek(int fd, off_t offset, int whence);
    • Return: 成功返回新的读写位置，失败返回-1
  2. 新的读写位置相对于whence参数：
    • SEEK_SET, SEEK_CUR, SEEK_END
  3. 得到当前的读写位置
    • off_t currpos = lseek(fd, 0, SEEK_CUR);
  4. 一些文件无法改变读写位置，如pipe/流文件(标准输入)，必须FIFO

a hole in a file

  1. lseek改变读写位置，中间可以跳过一定长度，这段长度填充为0
  2. 倘若读取没有写过的内容：
    • 按照字符读取，得到文件结束符
    • 按照无符号整数读取，得到0
  3. 空洞是否占用磁盘空间由文件系统决定

File Size

  1. st_size，stat结构成员，以字节为单位存储了文件带线啊哦
    • 对于符号链接，文件大小是指向文件的文件名的长度
  2. 如果将文件空洞读出来再拷贝，那么空洞不会被省略
    • 因为读的时候会用0填充

read/write

  1. ssize_t read(ing fd, void *buf, size_t nbytes);
    • Return: 读出来的文件长度，如果遇到EOF返回0，出错返回-1
    • 读取一个30个字节的文件，第一次读取100字节，返回30，第二次读返回0

IO Efficiency

  1. 每次读的数据对系统是有影响的
    • 每次读的数据越多，效率越高
    • 高到一定程度之后，就不会变化了
  2. LINUX中清除系统缓存的命令：
    • sync
    • echo 3 > /proc/sys/vm/drop_caches
      • 0：不释放（系统默认值）
      • 1：释放页缓存
      • 2：释放dentries和inodes
      • 3：释放所有缓存

Atomic Operations

  1. 在文件末尾写入，
    • 不合适：lseek–>write
    • 合适：O_APPEND，保证多个进程

pread

  1. ssize_t pread(int fd, void *buf, size_t nbytes, off_t offset);
    • Return: 成功返回读到的字节数，遇到EOF返回0，失败返回-1
  2. 封装了seek和read，定位到offset，读nbytes
  3. 真实的offset不会变化

Atomic Operations: create

  1. check and create
    • 不合适：open(O_WRONLY) –> ENOENT –> creat
    • 较为合适：open(O_CREAT | O_EXCL | O_RDWR) –> EEXIST –> open(O_RDWR)
      • 在EEXIT和第二次open间，文件可能会被删除，需要再次捕捉错误

dup

  1. j = dup(i)，复制i，j指向与i相同的file table项，共享读写模式和读写位置

other dup operation

  1. fcntl(fd, F_DUPFD, 0);
    • 对fd复制，返回新的fd
  2. dup2(fd, fd2);
    • 与dup功能一样，只是直接指定了新的文件描述符
    • 如果fd2本身有文件，则先close
    • 相当于：
      • close(fd2);
      • fcntl(fd, F_DUPFD, fd2);

redirect the i/o

  1. 相当于 $a.out > put0.txt

  2. fda = open(“put0.txt”, …);
    close(1);
    dup(fda);
    close(fda);
    printf(“…..”);

fcntl

  1. int fcntl(int fd, int cmd, …/* int arg */);

sync, fsync, fdatasync

  1. #include<unistd.h>
  2. int fsync(int fd);
    • 将缓冲区中所有有关fd的数据写入磁盘
    • 阻塞式调用，返回时确定已经写完了
  3. int fdatasync(int fd);
    • 将缓冲区中有关fd的数据写入磁盘，只写数据，不写元数据
  4. sync(void);
    • 将系统缓冲区中所有的dirty数据按照队列写入磁盘，然后返回
    • 只是写入工作开始，结束时间无法确定

ioctl

  1. int ioctl(int fd, int request, …);
  2. 包含了一些开发者自定义的，标准IO操作不包括的系统特性
    • eg：使用串口通信，还需要设置波特率等参数，不能通过标准读写设定
  3. 每一个驱动设备都可以自定义ioctl命令

/dev/fd

  1. /dev/fd/n，表示已经存在的fd
  2. fd = open(“/dev/fd/0”. mode);
    • 相当于 fd = dup(0);
