  • 课程目标：系统调用

  • Topics：

    • 文件系统/标准IO库
    • Unix进程环境和进程控制
    • 信号机制
    • 高级IO
      • 非阻塞IO，锁概念，流机制，IO服用，memory mapped IO

      • Pipes，Message Queues，Semaphores，Shared Memory

    • 并行程序设计
    • HADOOP架构

    • 高性能服务器软件架构分析

      • 系统性能测试与分析

  • POSIX：portable operating system interface

    • POSIX ≈ UNIX
    • Windows: win32 API

第一节

Files and Directories

  1. Filename
    • slash character (/) and the null character
  2. Working Directory
  3. Home Directory

Input and Output

  1. File Descriptors
  2. Standard Input, Output, Error –> 0, 1, 2
    • getchar()：从0文件描述符读取内容
  3. Standard I/O
    • 一个库，把底层的文件IO又进行一层包装

Threads and Thread IDs

  1. 线程是执行的实体，进程是资源的实体
    • 线程共享进程资源

Error Handling

  1. 底层的Error Handling基本和C语言是类似的
  2. 系统会通过调用的返回值
    • -1通常是有错误的，0通常是没错的
    • 系统会把错误信息存在环境变量errno中

User Identification

  1. User ID
    • 每一个用户在系统中都有一个用户ID，通过用户ID记录是哪一个用户在登录
    • 超户的ID是0
  2. group ID
    • 每一个用户都必须属于一个group
    • group的密码通常用不到，在设置group的权限时有用
    • /etc/group
    • 一个用户可以属于多个group
    • 其他的group叫做additional groups

Time Values

  1. Calendar time
  2. Process time(CPU time)
    • 进程在CPU上运行的时间
    • 计量单位 ticks
      • 一秒钟通常有 50/60/100 ticks
  3. 进程的运行时间有三种计量方式
    • Clock time(wall clock time)
      • 进程从被启动开始的运行时间，其中部分时间可能并不在CPU上运行
    • User CPU time
      • 进程在用户态执行时占据的CPU时间
    • System(Kernel) CPU time
      • 当我们调用系统调用或者终中断等在内核运行时占据的CPU时间
      • CPU time = User CPU time + System CPU time

System Calls and Library Functions

  1. System Calls
    • 让开发者可以通过这些系统调用接口来使用kernel提供的服务

UNIX standardization

  1. 相关的standard
    • interface
    • library/library location
    • data type
  2. 一些知名的标准
    • ANSI C
      • syntax，semantic，library
      • headers (stdio.h)
    • POSIX-操作系统标准
    • XPG4-欧洲的操作系统标准 (X/Open Portability Guide)
      • operating system interface standard
      • Defined in c
      • based on UNIX, but for all OS
  3. 标准规定了接口，但是没有规定实现，所以不同的厂商只要提供兼容的接口就行

UNIX implement

  1. SVR4 (system 5 release 4)
  2. 4.3+BSD
  3. Others(AIX, HP-UX, solaris, IRIX)
  4. LINUX conform to POSIX (LINUX 符合 POSIX标准)

Limits

  1. Compile-time limits (the largest value of a short integer)
    • 在头文件中define了宏，规定了使用的资源不能超过多少
  2. Runtime limits
    • 没有明确定义，但是使用过程中不能超过一定大小
    • eg：
      • 文件名长度，就的版本上不能超过 14 字符，新版本不能超过 255 字符
      • 进程打开文件的数目，不能超过一定数目
  3. 我们可以用一些接口来找到这些限制，也可在头文件中找到一些定义的宏来发现这些限制

ANSI C Limits

  1. 所有都是 compile-time 限制
    • eg：CHAR_BIT，限制了char的位

POSIX Limits

  1. compile-time
    • eg：_POSIX_LINK_MAX：number of links to a file 8
      • 这些值都是可以更改的

GET RUN Time limits

  1. #include <unistd.h>
    • long sysconf(int time)
    • long pathconf(const char *pathname, int time)
    • long fpathconf(int filedes, int name);
      • return: corresponding value if OK, -1 on error
      • 得到相应的限制

others

  1. OS version
    • #ifdef LINUX_2_4
  2. Platform
    • #ifdef INTEL
  3. software version
    • #ifdef DEBUG
  4. macro in command line
    • gcc -D DEBUG

Primitive System Data Types

  1. pid_t, dev_t, off_t
    • 本质上是证书，长整数等某一种类型，比如pid_t目前基本是int或者unsigned int类型
    • typedef __kernel_off_t off_t
  2. 写程序时，用到接口比如fork()，返回类型是pid_t，那么我们定义一个变量得到返回值时，也应该用pid_t
    • 与用int的区别：系统定义pid_t在不同系统或者在系统的不同时期可能是不同的类型，可能演变成一个数据结构
